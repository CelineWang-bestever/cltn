## 修复 customer-detail-pad.js 事件监听器提前绑定 Bug（9处按钮无响应）
- **根因**：`DOMContentLoaded` 回调中，事件绑定代码位于文件顶部（第12-198行），但绑定的 `window.xxxFunction` 均在更后面才赋值（第1433-2600行）。JS 中 `addEventListener('click', window.openMemberMoreInfo)` 在绑定时即求值为 `addEventListener('click', undefined)`，这是一个合法的 no-op，不会报错但也从不执行。
- **受影响的按钮**（共9处）：
  1. 更多信息按钮 → `window.openMemberMoreInfo` (defined l.1748)
  2. 补单弹窗-打开 → `window.openSupplementModal` (defined l.1459)
  3. 补单弹窗-关闭(X) → `window.closeSupplementModal` (defined l.1433)
  4. 补单弹窗-取消 → `window.closeSupplementModal` (defined l.1433)
  5. 补单弹窗-确认 → `window.confirmSupplement` (defined l.1437)
  6. 会员更多信息弹窗-关闭 → `window.closeMemberMoreInfo` (defined l.1768)
  7. 添加标签 → `window.addNewTag` (defined l.2492)
  8. 取消编辑 → `window.cancelMemberEdit` (defined l.2558)
  9. 保存会员信息 → `window.saveMemberInfo` (defined l.2600)
- **修复方案**：将 `addEventListener('click', window.xxxFn)` 改为 `addEventListener('click', function() { window.xxxFn(); })`，利用闭包的惰性求值特性——函数调用时才查找 `window.xxxFn`，此时已赋值完毕。

## 修复卡内剩余权益步进器 "+" 按钮失效
- **根因**：`drawer` 上绑定了 `e.stopPropagation()`（line 99），阻止所有点击事件冒泡到 `overlay`。而步进器事件委托绑在 `overlay.addEventListener('click', ...)`（line 114），事件永远无法到达，步进器完全不响应
- **修复**：将步进器事件委托从 `overlay` 移到 `drawer` 上，确保事件在 drawer 层级即可被捕获处理
- `stopPropagation` 保留不动，它仍负责阻止 drawer 内点击误关闭 overlay 遮罩
