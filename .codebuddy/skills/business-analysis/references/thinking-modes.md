# 八大思维方式详解

## 1. 闭环思维
从头至尾以闭环方式遍历所有步骤，确认需求分析没有遗漏。

**思考视角：**
- **用户视角**：以用户身份从头至尾走完整个流程
- **对象视角**：以对象的生命周期遍历所有环节

**应用案例：**
电商促销活动——仅从功能出发设计发优惠券流程，却忘了引流，造成活动失败。闭环思维可以发现引流等环节。

## 2. 对称思维
通过发现事情相反方向的场景确保需求分析完整性。

**不是左右对称**，而是通过相反方向探索更多可能性：
- 购买 ↔ 退货（全部退货 / 部分退货）
- 申请 ↔ 取消申请
- 签约 ↔ 解约 / 续约
- 汇款 ↔ 取消汇款

**注意事项：**
- 对于不准备应对的对称场景，明确说明"这里不对应"
- 不要混淆对称面和系统异常（异常场景参考异常思维）
- 影响性分析要附加影响性分析表

## 3. 边界思维
凡涉及时间、金额、距离、容量等可度量单位时，必须考虑边界。

**符号规范：**
- `[a, b]`：包含a和b的闭区间
- `(a, b]`：不包含a，包含b
- `>=` `<=` `>` `<`：数学符号表达

**分析维度：**
- 业务边界（业务规则限制）
- 系统边界（技术容量限制）

## 4. 异常思维
区分**业务错误**和**系统异常**：
- 业务错误：当前状态不允许执行某操作（正常逻辑的一部分）
- 系统异常：网络断开、磁盘不足等业务逻辑外的情况

**应用：**
- 网络异常 → 断线重连/离线模式
- 数据库异常 → 回滚+重试机制
- 第三方API超时 → 降级方案

## 5. 多样性思维
从以下维度探索多种可能性：

| 维度 | 示例 |
|------|------|
| 设备多样性 | 手机、平板、PC、刷卡机、蓝牙 |
| 角色多样性 | 普通会员/银卡/金卡/白金卡 |
| 业务种类多样性 | 自营仓库/租赁托管仓库 |
| 操作组合多样性 | Ctrl+S、多按钮组合 |
| 浏览器多样性 | Chrome/Firefox/Safari各版本 |
| 屏幕尺寸多样性 | 响应式适配 |

## 6. 生命周期思维
遍历对象生命周期各阶段，梳理：

| 阶段 | 梳理内容 |
|------|----------|
| 各生命周期节点 | 需要更新的字段列表 |
|    | 需要提供的功能 |
|    | 需要的外部依赖（其他系统、API） |

**示例（合同）：** 未签约 → 签约 → 履约 → 解约 → 续约

## 7. 并发思维
- 两人同时操作同一条数据 → 乐观锁/悲观锁
- 多用户同时提交 → 队列/限流
- 秒杀/抢购 → 库存扣减的原子性
- 性能压力预估（同时在线数、QPS）

## 8. 安全思维
- 角色权限：谁可以做什么
- 数据权限：谁能看到什么数据
- 防恶意操作：SQL注入、XSS、CSRF
- 敏感信息加密存储
- 审计日志
