# 需求分析工具速查表 (44 Tools Reference)

## 表达工具
| 工具 | 用途 |
|------|------|
| **时间轴** | 定义事件时间边界，梳理里程碑 |
| **表达式树** | 复杂逻辑判定的树形可视化 |
| **通用控件** | 统一描述搜索框、日期选择等，达到复用 |
| **数学公式** | 精确表达数学逻辑（如利息计算） |
| **概念图** | 表达抽象概念及其关系 |

## 结构设计工具
| 工具 | 用途 | 核心价值 |
|------|------|----------|
| **决策矩阵** | 状态×动作，判断在各状态下可执行的操作 | 100%覆盖，空白=Bug根源，引导面向对象编码 |
| **判定矩阵** | 条件笛卡尔积，判断条件组合下的结论 | Given-When-Then，可直接做测试用例 |
| **泳道图** | 跨角色流程可视化 | 分清各职能工作范围 |
| **网格式泳道图** | 泳道+时间轴组合 | 多角色多阶段流程 |
| **状态图** | 表达状态之间的关系和转移 | |
| **流程图** | 表达处理流程 | |
| **洋葱图** | 表达包含/层级关系 | |
| **探照灯** | 梳理状态列表 | |

## 逻辑工具
| 工具 | 用途 |
|------|------|
| **校验矩阵** | 字段|类型|必输|长度|格式|范围|唯一 的矩阵定义 |
| **API清单** | 请求方法、URL、请求/响应参数定义 |
| **定时任务** | Cron表达式 + 执行逻辑 + 异常处理 + 日志记录 |

## 设计分析方法
| 工具 | 用途 |
|------|------|
| **故事线法** | 以业务场景为主线，角色+泳道+界面缩略图，向业务方讲解 |
| **五步法** | 应对"一页纸需求"的完整分析方法 |
| **业务价值工作坊** | 群体决策，建立有效业务梳理机制 |
| **对称思维矩阵** | 对象|对称思维|影响性 的三列分析 |

## 建模工具
| 工具 | 用途 |
|------|------|
| **对象旅程** | 生命周期各阶段 → 字段 → 功能 → 外部依赖，避免遗漏 |
| **数据字典** | 类型|编码|值 的统一管理 |

## 非功能性需求
| 工具 | 用途 |
|------|------|
| **性能压力模板** | CPU/内存/带宽/并发数/响应速度的量化描述 |

## 工具选用决策速查

| 场景 | 推荐工具 |
|------|----------|
| 多条件组合判定 | 判定矩阵 |
| 有状态的对象操作 | 决策矩阵 |
| 跨角色流程 | 泳道图 |
| 时间范围定义 | 时间轴 |
| UI表单校验 | 校验矩阵 |
| 数据规范化 | 数据字典 |
| 外部系统梳理 | 对象旅程 |
| 向业务方展示 | 故事线法 |
| 复杂逻辑表达 | 表达式树 |
| 性能要求描述 | 性能压力模板 |
| 定时处理逻辑 | 定时任务模板 |
