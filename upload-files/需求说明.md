
# upload-files 需求说明

更新时间：2026-05-20  
覆盖范围：`upload-files/app-daily-upload-v2.html`、`upload-files/pad&pc-daily-records.html`

## 1. 项目概述

`upload-files` 目录承载“经营数据/资料上传与管理”的前端原型能力，包含两类入口形态：

- App 端上传页：用于移动端快速选择图片、补充可选命名、选择资料类型并提交上传（当前为模拟上传）。
- Pad&PC 端资料管理页：用于在桌面端对已上传图片进行筛选/排序/分页浏览，并支持查看详情、编辑元信息、删除图片，以及打开上传弹窗新增图片（当前为 mockData 模拟数据源）。

两端共享的核心概念包括：资料类型体系、可持久化的自定义资料类型标签、图片可选命名规则、图片列表的展示与删除等。

## 2. 功能需求清单

### 2.1 App 上传页（app-daily-upload-v2.html）

- 资料类型选择：系统类型 + 自定义标签（可新增、去重、持久化）。
- 资料类型标签收起/展开：当标签内容行数超过 3 行时默认收起，仅展示前 3 行；支持“查看更多/收起”切换。
- 自定义标签数量上限：最多可自定义 15 个；达到上限后隐藏输入区并提示“无法添加新标签”。
- 图片添加：拍照上传、相册选择（多选）。
- 图片列表：缩略图、状态标识、删除按钮、可选“图片命名”输入。
- 上传提交：校验后发起上传（当前为模拟），展示 toast 状态并在成功后清空待上传列表。
- 离开确认：存在未上传图片时返回会二次确认。

### 2.2 Pad&PC 管理页（pad&pc-daily-records.html）

- 检索筛选：
  - 资料类型：全部/各系统类型/自定义标签（标签式筛选，动态渲染）。
  - 资料类型标签收起/展开：默认最多展示 1 行，超出后显示“查看更多/收起”切换。
  - 上传日期范围：开始/结束日期。
  - 上传人：下拉选择。
  - 图片名称：关键字匹配（同时支持大小写不敏感匹配）。
  - 支持“搜索/重置”。
- 排序：
  - 排序字段：按上传时间 / 按图片名称 / 按修改时间。
  - 排序方向：正序/倒序切换。
- 图片平铺展示 + 分页：
  - 每页 12 张图片。
  - 页码按钮 + 上一页/下一页。
  - 空数据时展示空态。
- 图片查看弹窗（详情弹窗）：
  - 左侧大图预览，右侧元信息面板。
  - 支持上一张/下一张切换（按钮 + 键盘左右方向键）。
  - 支持进入编辑态：修改资料类型（含自定义标签）、修改图片命名（选填）并保存。
  - 支持删除当前图片（弹出删除确认弹窗）。
- 上传弹窗：
  - 复用 App 上传页的上传 UI（资料类型 + 自定义标签 + 图片选择 + 命名 + 提交）。
  - 提交上传后模拟进度，成功后将数据写入 `mockData` 并刷新列表。
- 自定义标签管理：
  - 入口位于页面操作条中，位于“上传”按钮左侧。
  - 支持删除未被图片使用的自定义标签；已被图片使用的标签禁止删除。

## 3. 页面交互流程

### 3.1 App 上传页流程

1. 进入页面后默认选中资料类型“每日经营”。
2. 用户可在“资料类型”区域：
   - 点击系统类型标签切换；
   - 输入自定义标签并点击“添加”，新增后自动选中；
   - 自定义标签会持久化到 localStorage。
   - 自定义标签达到 15 个上限后：输入区隐藏，并提示“您已定义15个自定义标签，无法添加新标签”。
3. 资料类型标签行数超过 3 行时默认收起，点击“查看更多/收起”切换显示全部/仅前三行。
4. 页面滚动：资料类型标签区域随页面正常滚动，不再吸顶固定。
5. 用户通过“拍照上传 / 相册选择”添加图片（多选）：
   - 超过上限或格式不支持会 toast 提示；
   - 同名同大小文件视为重复，会被静默去重；
   - 每张图片生成缩略图并展示在“已上传”区域。
6. 用户可为每张图片输入“图片命名（选填）”，也可点击清空按钮一键清空该图片的命名。
7. 点击“确认上传”：
   - 若无图片，提示“请先选择文件”；
   - 否则 toast “正在上传...”，1.5s 后 toast “上传成功”，并清空待上传列表（模拟上传完成）。
8. 点击返回：
   - 若存在待上传图片，会弹出浏览器 confirm 二次确认。

### 3.2 Pad&PC 管理页流程

#### 3.2.1 筛选/排序/分页浏览

1. 页面加载后渲染图片平铺列表（从 `mockData` 构建图片级 item 列表）。
2. 资料类型筛选标签默认展示 1 行，若超出则显示“查看更多/收起”以展开或收起多行标签。
3. 点击资料类型标签（包含自定义标签）触发筛选，与系统标签的交互逻辑一致。
4. 用户设置筛选条件后点击“搜索”，会刷新列表并 toast “搜索完成”。
3. 点击“重置”恢复默认条件并刷新列表，toast “已重置筛选条件”。
4. 修改排序字段或排序方向，会回到第 1 页并刷新列表。
5. 点击分页按钮切换页码，刷新当前页图片列表。

#### 3.2.2 查看图片详情 / 上一张下一张

1. 点击任意缩略图进入详情弹窗。
2. 弹窗左侧展示大图，右侧展示元信息：
   - 资料类型、自定义命名、上传时间、上传人、修改时间、修改人。
3. 可通过：
   - “上一张/下一张”按钮切换；
   - 键盘 ← / → 切换（当焦点不在输入框/文本域时）。
4. 关闭弹窗（点击遮罩或关闭按钮）后，清空当前详情定位。

#### 3.2.3 详情内编辑与保存

1. 点击“编辑”进入编辑态：
   - 可修改资料类型：系统类型 + 自定义标签（同样支持新增/去重/持久化）。
   - 可修改图片命名（选填）。
2. 点击“保存”：
   - 将 `customName/type/typeName/modifiedAt/modifiedBy` 写回对应 file 元数据；
   - 刷新右侧元信息展示；
   - 刷新图片平铺列表（名称、类型筛选等即时生效）；
   - toast “保存成功”。
3. 点击“取消”退出编辑态，不写回。

#### 3.2.4 删除图片

1. 在详情弹窗点击“删除”，弹出删除确认弹窗。
2. 点击确认后删除对应图片：
   - 若图片 thumb 为 blob URL，会先 revoke；
   - 删除后若该 record 无图片，则移除整个 record；
   - 刷新列表并 toast “删除成功”。

#### 3.2.5 上传弹窗（Pad&PC）

1. 点击页面右上角“上传”打开上传弹窗：
   - 初始化为清空已选图片；
   - 载入自定义标签并渲染类型选项。
2. 通过“拍照上传/相册选择”添加图片：
   - 上限 12；
   - 支持扩展名校验：jpg/jpeg/png；
   - 同名同大小去重；
   - 生成 blob 缩略图并展示。
3. 点击“确认上传”：
   - 若未选择资料类型，提示“请选择资料类型”；
   - 若未选择图片，提示“请先选择图片”；
   - 否则将所有待上传图片置为 uploading 状态并显示按钮进度（模拟）。
4. 上传成功后：
   - 生成一条新的 record 写入 `mockData`（包含文件级元信息）；
   - 刷新图片列表；
   - 关闭上传弹窗并保留本次上传生成的 blob URL（避免预览失效）。

#### 3.2.6 自定义标签管理（Pad&PC）（本期不实现）

1. 点击操作条中“自定义标签管理”打开管理弹窗。
2. 管理弹窗展示：
   - 自定义标签名称；
   - 已关联图片数量（用于判断是否可删除）。
3. 删除：
   - 点击“删除”触发二次确认；
   - 若该标签已被图片使用则禁止删除（按钮禁用且提示）；
   - 删除成功后将更新 localStorage，并在自定义标签数量低于 15 个时自动恢复输入区可用。

## 4. 输入输出规范（字段/数据结构）

### 4.1 资料类型体系

- 系统资料类型（两页一致）：
  - daily：每日经营
  - customer：客户资料
  - appointment：预约表
  - kpi：员工月度KPI
  - income：收入及消费分析
  - product：品项经营分析
  - financial：财务支出
  - other：其他
- 自定义资料类型：
  - key 形态：`custom:${label}`
  - label 最大长度：20
  - label 数量上限：15
  - label 过滤规则：
    - 去除控制字符（U+0000 ~ U+001F）
    - 去除非法字符：`\\ / : * ? " < > |`
    - 连续空白归一化为单空格并 trim

### 4.2 localStorage

- `daily_upload_custom_types_v2`：
  - 类型：JSON 数组（字符串数组）
  - 含义：用户创建的自定义资料类型标签列表
  - 使用页面：App 上传页、Pad&PC 管理页（上传弹窗与详情编辑均会使用）

### 4.3 App 上传页：待上传图片结构（前端内存态）

- `uploadedFiles[]` item 结构（用于渲染“已上传”网格）：
  - `id`: number（`Date.now() + i`）
  - `name`: string（原文件名）
  - `customName`: string（用户输入的图片命名，选填）
  - `file`: File
  - `thumb`: string（`URL.createObjectURL(file)`）
  - `status`: 'success' | 'error' | 'uploading'（当前实现默认 success）
  - `size`: number（byte）

### 4.4 Pad&PC 管理页：记录与文件结构（mockData）

- `mockData[]` record 结构（示例/模拟数据源）：
  - `id`: number
  - `type`: string（资料类型 key）
  - `typeName`: string（资料类型展示名）
  - `businessDate`: string（YYYY-MM-DD，当前上传生成取当天）
  - `uploader`: string（上传人，mock 中为示例或“当前用户”）
  - `time`: string（上传时间，YYYY-MM-DD HH:mm）
  - `description`: string（描述/备注，当前上传生成为空）
  - `files[]`: file 元信息数组

- `files[]` 内 file 元信息（图片条目关注字段）：
  - `filename`: string（展示名；保存编辑时可能被覆盖）
  - `customName`: string（自定义命名，选填）
  - `size`: string（展示用，如 1.2MB / 856KB / '—'）
  - `thumb`: string | null（图片缩略图；非图片文件可能为 null）
  - `type`: string（资料类型 key）
  - `typeName`: string（资料类型展示名）
  - `uploadTime`: string
  - `modifiedAt`: string
  - `modifiedBy`: string

## 5. 业务规则与约束

### 5.1 图片选择与校验（两端共性）

- 数量上限：最多 12 张（超过提示并停止继续添加）。
- 支持格式：
  - App 上传页：通过扩展名校验 `jpg/jpeg/png`。
  - Pad&PC 上传弹窗：通过扩展名校验 `jpg/jpeg/png`。
- 去重规则：同名 + 同 size 的文件视为重复，重复项会被静默忽略。

### 5.2 图片命名（两端共性）

- 字段：`customName`（选填）。
- 最大长度：50。
- 过滤规则：
  - 去除控制字符（U+0000 ~ U+001F）
  - 去除非法字符：`\\ / : * ? " < > |`
- 展示名策略（Pad&PC 管理页）：
  - 优先展示 `customName`，为空则展示 `filename`。

### 5.3 Blob URL 生命周期（Pad&PC 上传弹窗/删除）

- 上传弹窗打开时会清理上一次残留的 blob URL（防内存泄漏）。
- 上传弹窗关闭时：
  - 默认会 revoke 本次选择产生的 blob URL；
  - 若为“上传成功后关闭”，会以 `closeUploadModal(true)` 保留 blob URL，避免上传成功后列表预览失效。
- 删除图片时：若 thumb 为 blob URL，会 revoke。

### 5.4 详情弹窗导航规则（Pad&PC）

- 当前图片在“筛选 + 排序”的结果集中定位 index。
- 上一张/下一张按钮：
  - 首张禁用“上一张”，末张禁用“下一张”。
- 键盘左右方向键：
  - 仅在详情弹窗开启且焦点不在输入框/文本域时生效。

### 5.5 可用性与视觉设计要点（从实现抽取）

- App 上传页：
  - `max-width: 430px`，强调移动端单列布局；
  - 资料类型选择器随页面正常滚动（不吸顶）；
  - 资料类型标签超过 3 行时默认收起，提供“查看更多/收起”入口；
  - 自定义标签输入区下方提示“最多可自定义15个标签”，达到上限后输入区隐藏并展示占位提示；
  - 底部固定操作区：拍照/相册按钮 + 确认上传按钮；
  - 已上传区为 3 列网格展示（缩略图 + 删除）。
- Pad&PC 管理页：
  - 顶部筛选卡片（类型标签 + 多条件筛选）；
  - 顶部类型标签支持自定义标签动态渲染，默认最多展示 1 行并提供“查看更多/收起”；
  - 中部操作条（排序控制 + 上传入口）；
  - 操作条右侧包含“自定义标签管理”与“上传”按钮（管理在左、上传在右）；
  - 图片平铺展示区（缩略图点击进入详情）；
  - 详情弹窗右侧采用“查看/编辑面板栈”切换，底部按钮区根据状态切换“编辑/删除”或“取消/保存”。
  - 二次确认弹窗（删除图片/删除自定义标签）需保证 z-index 在所有弹窗之上，避免被遮挡。

## 6. 测试用例设计说明

### 6.1 App 上传页（app-daily-upload-v2.html）

- 用例 A1：新增自定义资料类型
  - 操作：输入自定义标签（含空格、非法字符、超长），点击添加
  - 预期：标签被 sanitize；重复标签提示“标签已存在”；新增后自动选中；刷新后仍存在（localStorage）
- 用例 A2：图片选择校验
  - 操作：一次选择超过 12 张；选择非 jpg/png 图片
  - 预期：超过上限提示“最多上传12个文件”；不支持格式提示“xxx 格式不支持”；重复文件不重复加入
- 用例 A3：图片命名校验
  - 操作：输入超长 50、非法字符、控制字符
  - 预期：输入被实时 sanitize，长度被裁剪；清空按钮生效
- 用例 A4：提交上传（模拟）
  - 操作：无图点击确认；有图点击确认
  - 预期：无图提示；有图提示上传中→成功，列表清空，按钮禁用恢复
- 用例 A5：返回确认
  - 操作：有待上传图片时点击返回
  - 预期：弹出二次确认
- 用例 A6：资料类型标签收起/展开
  - 操作：构造资料类型标签为 2/3/4 行及以上，观察默认展示并点击“查看更多/收起”
  - 预期：>3 行时默认只展示前 3 行，点击后可展开全部并可再次收起；<=3 行不展示入口
- 用例 A7：自定义标签数量上限（15）
  - 操作：连续新增自定义标签直到 15 个，再尝试新增第 16 个
  - 预期：达到 15 个时输入区隐藏并展示“无法添加新标签”；新增第 16 个被阻止并提示

### 6.2 Pad&PC 管理页（pad&pc-daily-records.html）

- 用例 P1：筛选与搜索
  - 操作：按类型筛选、设置日期范围、选择上传人、输入图片名称关键字，点击搜索
  - 预期：列表仅展示匹配图片，分页与总数更新，toast“搜索完成”
- 用例 P2：排序与分页
  - 操作：切换排序字段与方向；翻页
  - 预期：顺序变化符合规则；分页按钮启用/禁用正确
- 用例 P3：详情查看与键盘切换
  - 操作：点击缩略图打开详情；使用按钮/键盘左右切换
  - 预期：元信息随图片更新；首末禁用状态正确；输入框聚焦时键盘切换不触发
- 用例 P4：详情编辑保存
  - 操作：进入编辑态，修改类型/命名并保存
  - 预期：右侧元信息更新；平铺列表名称/类型筛选结果同步变化；toast“保存成功”
- 用例 P5：删除图片
  - 操作：详情内删除，确认
  - 预期：图片从列表移除；若 record 无图片则 record 被移除；toast“删除成功”
- 用例 P6：上传弹窗添加图片
  - 操作：打开上传弹窗，通过按钮选择图片，填写命名，确认上传
  - 预期：数量/格式/去重校验生效；模拟进度后列表新增图片；关闭弹窗后新增图片仍可预览（blob URL 未被 revoke）
- 用例 P7：顶部资料类型筛选（含自定义标签）+ 查看更多
  - 操作：创建多条自定义标签并在顶部筛选区点击自定义标签筛选；当标签超过 1 行时点击“查看更多/收起”
  - 预期：自定义标签与系统标签筛选行为一致；>1 行时默认收起到 1 行并可展开/收起；筛选状态切换不影响原有功能
- 用例 P8：自定义标签管理（禁删占用 + 删除未占用）
  - 操作：对已被图片使用的自定义标签尝试删除；对未被使用的自定义标签删除并确认
  - 预期：已使用标签删除按钮禁用且提示无法删除；未使用标签可删除，删除后自定义标签数量 <15 时输入区恢复可用
