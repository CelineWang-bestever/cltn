# 费用支出记账 — 需求文档

## 1. 需求概述

| 项目       | 内容                                                                                                       |
| -------- | -------------------------------------------------------------------------------------------------------- |
| **需求类型** | 新增需求 — 门店日常支出记账管理                                                                                        |
| **业务背景** | 门店在日常经营中产生各类费用支出（房租、工资、耗材、营销等）。店长/店员需要一套工具来记录、查询和统计这些支出，为后续经营损益分析提供基础数据。                                 |
| **核心目标** | 1. 提供统一的支出记账入口，覆盖门店经营中各类费用类别；2. 支持自定义支出类别和项目，灵活适应不同门店的财务核算需求；3. 提供筛选、搜索、分页能力，便于查阅历史记录；4. 支持数据导出，与损益报表联动。 |

## 3. 术语定义

| 术语      | 英文               | 说明                                                                 |
| ------- | ---------------- | ------------------------------------------------------------------ |
| 支出类别    | Expense Category | 支出的第一层分类，如"日常运营成本"、"人员支出"、"产品耗材"、"设备相关"、"营销推广"、"其他费用"              |
| 具体项目    | Expense Item     | 隶属于支出类别下的具体支出项，如"房租"、"工资"、"社保"等                                    |
| 自定义类别   | Custom Category  | 用户自行新增的支出类别，非系统预设                                                  |
| 自定义项目   | Custom Item      | 用户自行新增的支出项目，隶属于某个类别                                                |
| 货币单位    | Currency         | 默认币种为人民币 CNY，金额按元展示并统一保留 2 位小数                                     |
| 名称变更规则  | Rename Rule      | 编辑类别/项目名称后，历史支出记录中的类别/项目名称同步更新为最新名称                                 |
| 删除规则    | Delete Rule      | 系统预设类别/项目与自定义类别/项目均支持删除，但若已有支出记录关联，则禁止删除，需先删除或转移相关支出记录              |
| 经办人     | Handler / Person | 实际办理该笔支出的人员                                                        |
| 版本号     | Version          | 用于乐观锁并发控制的数据版本字段，后端校验时以最新版本为准                                       |

## 4. 业务流程分析

### 4.1 主流程 — 新增一笔支出

```
[店长/店员] → 点击"记一笔" → 弹出新增支出弹窗
   → 搜索已有项目（可选）
   → 选择/切换支出类别（左侧栏）
   → 选择具体项目（右侧网格，单选）
   → 填写表单：
       - 实际支出金额（必填，默认币种人民币 CNY，统一保留 2 位小数）
       - 支出日期（默认当天，可改）
       - 经办人（下拉选择）
       - 备注（选填）
   → 点击"保存"
   → 弹窗关闭，列表刷新
```

### 4.2 子流程 — 新增自定义类别

```
[新增支出弹窗中]
   → 类别列表底部 → 点击"自定义类别"
   → 输入类别名称
   → 点击确认 ✓ 或键盘Enter
   → 验证：名称不允许为空，不允许与已有类别重名
   → 新类别创建，自动选中，右侧项目列表为空
   → 该项目类别下无预设项目，需手动添加
```

### 4.3 子流程 — 新增自定义项目

```
[新增支出弹窗中]
   → 右侧项目网格底部 → 点击"自定义项目"
   → 输入项目名称
   → 点击确认 ✓ 或键盘Enter
   → 验证：名称不允许为空，不允许与当前类别下已有项目重名
   → 新项目添加到当前类别，网格刷新
```

### 4.4 子流程 — 编辑类别/项目名称

```
[类别/项目管理场景]
   → 点击编辑入口
   → 输入新的类别名称或项目名称
   → 验证：名称不允许为空，不允许与同级已有名称重名
   → 保存成功后，历史支出记录中的类别/项目名称同步更新为最新名称
```

### 4.5 子流程 — 删除类别/项目

```
[新增支出弹窗中]
   → 鼠标悬停类别项或项目项 → 显示删除按钮
   → 点击删除按钮
   → 确认对话框
   → 验证：至少保留1个类别，否则不允许删除类别
   → 后端校验：若类别/项目已被支出记录引用，则提示需先删除或转移相关支出记录，不允许删除
   → 若无关联支出记录，则允许删除；删除类别时同步删除旗下全部项目
   → 筛选下拉框同步更新
```

### 4.6 子流程 — 搜索项目

```
[新增支出弹窗中]
   → 在顶部搜索框输入关键词
   → 实时过滤：在所有类别下匹配项目名称（大小写不敏感）
   → 匹配到项目的类别保持高亮，无匹配的类别半透明（opacity:0.3）
   → 右侧项目列表仅显示匹配到的项目
   → 右侧标题显示"搜索结果（N 个匹配）"
   → 清空搜索框 → 恢复默认视图
```

### 4.7 子流程 — 列表筛选

```
[主页面支出明细]
   → 搜索框：输入关键词过滤项目名
   → 类别筛选下拉：单选某一类别
   → 项目筛选下拉：单选某一项目（受类别筛选联动影响）
   → 日期范围筛选：默认当前月 1 号到当前月最后一天，可手动修改起止日期
   → 点击"筛选" → 应用所有条件
   → 页面联动刷新“筛选结果统计”（支出合计、支出笔数）
   → 点击"重置" → 清空所有条件
```

### 4.8 子流程 — 编辑/删除记录

```
[主页面支出明细表格]
   → 每行操作列：
       - 编辑按钮（铅笔图标）：进入编辑模式（复用新增弹窗，回填数据）
           → 允许修改类别、项目、金额、日期、经办人、备注
           → 保存前校验目标类别/项目必须存在
           → 若目标类别/项目已被删除，则提示并阻止保存
       - 删除按钮（垃圾桶图标）：确认后删除该条记录
```

### 4.9 子流程 — 导出

```
[主页面]
   → 点击"导出"按钮
   → 按当前筛选条件导出全部结果数据为XLS文件（不受当前分页限制）
   → 导出列包含：支出类别、具体项目、实际支出、支出日期、经办人、备注
   → 文件名命名规则：支出明细_YYYY-MM-DD.xls（示例：支出明细_2026-06-12.xls）
```

## 5. 判定矩阵

### 5.1 新增类别 — 条件判定

| 名称是否为空 | 名称是否已存在 | 结果                |
| ------ | ------- | ----------------- |
| 是      | —       | 不允许保存，提示"请输入类别名称" |
| 否      | 是       | 不允许保存，提示"该类别已存在"  |
| 否      | 否       | 创建成功，自动选中新类别      |

### 5.2 新增项目 — 条件判定

| 类别是否存在 | 名称是否为空 | 名称是否已存在 | 结果                |
| ------ | ------ | ------- | ----------------- |
| 否      | —      | —       | 不允许操作（异常情况）       |
| 是      | 是      | —       | 不允许保存，提示"请输入项目名称" |
| 是      | 否      | 是       | 不允许保存，提示"该项目已存在"  |
| 是      | 否      | 否       | 创建成功，网格刷新         |

### 5.3 删除类别/项目 — 条件判定

| 是否删除类别 | 当前类别总数 | 是否存在关联支出记录 | 操作                          |
| ------ | ------ | ---------- | --------------------------- |
| 是      | <= 1   | —          | 不允许删除，提示"至少保留一个类别"          |
| 是/否    | > 1    | 是          | 不允许删除，提示"请先删除或转移相关支出记录"     |
| 是/否    | > 1    | 否          | 弹出确认框，确认后删除；删除类别时同步删除旗下全部项目 |

### 5.4 编辑支出记录 — 类别/项目校验

| 目标类别是否存在 | 目标项目是否存在 | 结果                             |
| -------- | -------- | ------------------------------ |
| 否        | —        | 不允许保存，提示"目标类别不存在或已被删除，请刷新后重试" |
| 是        | 否        | 不允许保存，提示"目标项目不存在或已被删除，请刷新后重试" |
| 是        | 是        | 允许保存                           |

## 6. 关键场景覆盖

### 6.1 正向场景

- 店长打开页面，查看到统计卡片（本月总支出、本月支出笔数、最大单笔、今日支出）
- 查看支出明细列表，支持分页浏览
- 调整搜索、类别、项目、日期范围等筛选条件后，页面展示“筛选结果统计”，联动显示当前结果的支出合计与支出笔数
- 点击"记一笔"，从预设类别和项目中勾选，填写金额和日期，保存成功
- 编辑已有支出记录时，可修改类别和项目；若目标类别/项目已被删除，则提示并阻止保存
- 点击"导出"后，按当前筛选条件导出全部结果数据，导出列包含备注，文件名按日期自动命名

### 6.2 对称场景（逆向操作）

- 新增支出 ↔ 删除支出
- 新增类别 ↔ 删除类别
- 新增项目 ↔ 删除项目
- 编辑支出（覆盖原记录）

### 6.3 边界条件

- 金额：输入值应 > 0，按元存储并统一保留 2 位小数；若输入超过 2 位小数（如粘贴 `12.345`），则按四舍五入处理为 `12.35`
- 列表筛选日期：默认值为当前月 1 号到当前月最后一天，用户可手动修改为任意日期范围
- 类别数量：最少保留1个类别，无上限
- 项目数量：每个类别下无上限
- 今日支出：按自然日 00:00-23:59 统计
- 分页：列表超过一页时显示分页控件，单页 10 条记录

### 6.4 异常处理

| 异常场景             | 处理方案                                                                 |
| ---------------- | -------------------------------------------------------------------- |
| 未选择项目就保存         | 应拦截，提示"请选择支出项目"                                                     |
| 未填写金额就保存         | 应拦截，提示"请输入实际支出金额"；若金额 <= 0，则提示"请输入大于 0 的实际支出金额"；若两项都缺，优先提示"请选择支出项目" |
| 编辑记录时目标类别已被删除    | 提示"目标类别不存在或已被删除，请刷新后重试"，阻止保存                                        |
| 编辑记录时目标项目已被删除    | 提示"目标项目不存在或已被删除，请刷新后重试"，阻止保存                                        |
| 删除类别/项目但已有支出记录关联 | 提示"请先删除或转移相关支出记录"，不允许删除                                              |
| 多人同时新增/删除类别或项目   | 采用后端乐观锁（基于版本号）校验；以后端结果为准，冲突时提示"数据已被修改，请刷新"并返回明确错误码                  |
| 搜索无匹配结果          | 显示"未找到匹配项目"                                                         |

## 8. 数据模型

### 8.1 支出类别

| 预设类别   | 预设项目                                 |
| ------ | ------------------------------------ |
| 日常运营成本 | 房租、物业费、水费、电费、网络通讯费、清洁耗材、其他（鲜花/水果/茶水） |
| 人员支出   | 工资、社保、公积金、培训费用、节日福利、生日礼金、生育期薪资       |
| 产品耗材   | 美容产品、仪器耗材                            |
| 设备相关   | 仪器折旧费、设备维护费、新设备购置                    |
| 营销推广   | 线上推广（美团/抖音）、线下活动物料、客户礼品              |
| 其他费用   | 税费、工商年检费、基础设施维修/更换                   |

### 8.2 支出记录

| 字段     | 类型     | 必填 | 说明                     |
| ------ | ------ | -- | ---------------------- |
| 支出类别   | 枚举/外键  | 是  | 关联到支出类别                |
| 具体项目   | 枚举/外键  | 是  | 关联到具体项目                |
| 实际支出金额 | 数值(元)  | 是  | 默认币种人民币 CNY；精确到分，统一保留 2 位小数，范围 > 0；超过 2 位小数按四舍五入处理 |
| 支出日期   | 日期     | 是  | 默认当天，可修改               |
| 经办人    | 外键/字符串 | 是  | 默认当前登录员工，可修改，从员工列表选择   |
| 备注     | 文本     | 否  | 自由文本，最大上限200个字符        |
| version  | 整数     | 是  | 乐观锁版本号，用于并发校验          |

## 9. UI设计要点

### 9.1 页面布局

- 顶部导航栏：品牌名 + 门店名 + 用户信息
- 页面标题：费用支出记账 + 操作按钮（导出、记一笔）
- 统计卡片行：4个卡片（本月总支出、本月支出笔数、最大单笔、今日支出），展示全店本月数据，不受列表筛选影响
- 支出明细面板：搜索 + 筛选 + 筛选结果统计 + 表格 + 分页

### 9.2 弹窗布局

- 顶部标题："新增支出" / "编辑支出"
- 搜索条：实时搜索项目
- 左右分栏：左侧类别列表 + 右侧项目网格
- 底部表单：金额、日期、经办人、备注
- 底部操作：取消、保存

### 9.3 筛选类别联动规则

- 选择类别后，项目下拉仅显示该类别下的项目
- 类别为"全部"时，项目下拉显示全部项目
- 切换类别时自动清空项目选择

### 9.4 金额显示规范

- 表格中金额右对齐
- 显示格式：¥ 1,000.00（千位分隔符 + 保留 2 位小数）
- 货币单位：默认人民币 CNY
- 输入超过 2 位小数时按四舍五入处理

### 9.5 筛选结果统计

- 位置：位于筛选区下方、表格上方
- 内容：展示“支出合计”和“支出笔数”两项统计
- 统计口径：基于当前筛选后仍可见的表格记录实时计算，其中支出合计统一保留 2 位小数
- 联动条件：搜索、类别、项目、起止日期变化后同步刷新
- 与统计看板关系：筛选结果统计仅反映当前列表结果；统计看板始终展示全店全局数据，不受筛选影响
