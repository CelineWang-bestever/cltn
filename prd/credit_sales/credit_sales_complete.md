# 赊销铺货（Credit Sales）完整需求文档

---

## 1. 产品概述

### 1.1 产品背景

赊销铺货是美容行业供应链管理的重要模式，指供应商先将商品铺货到门店销售，待商品销售后再与供应商结算货款。克丽缇娜作为美容行业领先品牌，需要一套完整的赊销铺货管理系统，实现合同管理、铺货额度控制、销售数据同步、结算对账等核心业务流程的数字化管理。

### 1.2 产品目标

- 实现赊销铺货业务的全流程数字化管理
- 支持批次化的门店签约管理
- 实现铺货额度的实时控制和预警
- 提供完整的结算对账功能
- 支持多维度数据分析和报表
- 提升门店与总部的协同效率

### 1.3 适用范围

| 对象 | 角色 | 使用设备 | 核心场景 |
|------|------|----------|----------|
| 总部管理 | 运营管理员、财务人员 | PC端 | 批次管理、结算审核、数据分析 |
| 门店端 | 店长、店员 | PC端、Pad端 | 合同签署、商品管理、结算发起 |

## 3. 功能需求

### 3.1 管理后台端功能

#### 3.1.1 批次管理

**批次列表**
- 显示所有报名批次信息
- 支持按批次名称搜索筛选
- 显示批次编号、名称、合同起止时间、签约门店数、未签约门店数

**批次操作**
- 新增报名批次：设置批次名称、开始时间、结束时间
- 编辑批次信息
- 删除批次（仅限未签约门店的批次）
- 查看批次详情
- 查看门店数据

**门店批量导入**
- 支持 Excel 文件导入
- 模板下载和格式校验
- 导入结果反馈

#### 3.1.2 门店管理

**门店列表**
- 显示所有寄售门店信息
- 支持按区域、状态筛选
- 显示门店名称、联系人、联系方式、合同状态

**门店状态管理**
- 门店状态：待审核、正常合作、暂停合作、终止合作
- 状态变更记录
- 暂停/终止原因记录

#### 3.1.3 结算单管理

**结算单列表**
- 显示所有结算单
- 支持按时间、门店、状态筛选
- 显示结算单号、门店名称、结算金额、状态

**结算单详情**
- 结算基本信息
- 商品明细列表
- 金额汇总
- 操作记录

**结算操作**
- 审核结算单
- 生成结算凭证
- 导出结算数据

### 3.2 PCStore 端功能

#### 3.2.1 合同管理

**合同签署**
- 合同待签署提醒
- 合同条款查看

**合同详情**

- 合同文本快照

## 4. 业务流程

### 4.1 批次管理流程

```
创建批次 → 设置时间范围 → 关联门店 → 门店签约 → 合同生效 → 铺货发货 → 销售结算
```

### 4.2 合同签署流程

```
系统推送通知 → 门店查看合同 → 确认条款 → 电子签署 → 合同生效 → 开始铺货
```

### 4.3 结算对账流程

```
生成对账单 → 门店确认 → 财务审核 → 确认结算金额 → 生成结算单 → 发起付款 → 付款完成 → 核销归档
```

### 4.4 合同续约流程

```
合同到期提醒 → 新旧条款对比 → 确认续约 → 电子签署 → 新合同生效
```

## 8. 非功能性需求

### 8.1 性能要求

- 页面加载时间 ≤ 2秒
- 数据查询响应时间 ≤ 1秒
- 支持1000+批次、10000+门店数据管理
- 支持100+并发用户

### 8.2 兼容性要求

- 支持 Chrome、Firefox、Safari、Edge 最新版本
- 适配 1920×1080 及以上分辨率
- 移动端适配 iOS 和 Android

### 8.3 安全要求

- 用户身份认证和权限控制
- 操作日志完整记录
- 合同文件安全存储
- 敏感数据加密传输

### 8.4 可用性要求

- 系统可用性 ≥ 99.9%
- 数据定期备份（每日备份、异地备份）
- 故障恢复时间 ≤ 30分钟

## 10. 附录

### 10.1 合同签署通知规则

系统将于合同生效开始日期前15天推送签署合同通知，不足15天将于次日上午8:30推送通知。

### 10.2 结算周期

- 默认结算周期：45天
- 逾期滞纳金：按日万分之五计算

### 10.3 数据导出格式

支持 Excel 格式导出（.xlsx, .xls）
