# 12 - 支付页：第三方渠道收款

> **所属模块**: 支付页（`new-order-design/payment.html`）
> **核心程度**: ⭐⭐⭐ 核心

---

## 背景与目标

在支付页新增一组“第三方渠道收款”收款方式，用于处理来自抖音/美团等第三方平台的券码核销场景。该场景与“组合收款”互斥：当用户选择第三方渠道时，仅允许单一渠道收款，并要求录入券码（支持手动输入或摄像头扫码填充）。

---

## 功能范围

### 12.1 新增收款分组：第三方渠道收款

| 属性 | 说明 |
|-----|------|
| 分组标题 | 第三方渠道收款 |
| 选项 | 抖音 / 美团/点评 / 线下 |
| 展示位置 | 支付页左侧“添加收款方式”列表 |

---

### 12.2 不支持组合收款（与组合开关互斥）

| 属性 | 说明 |
|-----|------|
| 互斥规则 | 选择第三方渠道时，强制关闭组合收款 |
| 开关行为 | 若当前为第三方渠道收款，点击“组合收款”开关提示“不支持组合收款”，开关保持关闭 |

---

### 12.3 右侧输入区改为券码录入

| 属性 | 说明 |
|-----|------|
| 输入内容 | 券码（placeholder：请输入券码） |
| 输入方式 | 手动输入 + 扫码核销 |
| 交互 | 切换到非第三方渠道时，右侧恢复为原有金额输入逻辑 |

---

### 12.4 扫码核销（摄像头）

| 属性 | 说明 |
|-----|------|
| 打开方式 | 点击“扫码核销”按钮 |
| 能力依赖 | `navigator.mediaDevices.getUserMedia` + `BarcodeDetector`（QR Code） |
| 成功结果 | 识别到二维码后自动填充券码并关闭扫码弹窗 |
| 异常降级 | 无摄像头/无权限/不支持识别：提示用户手动输入券码 |

---

## 数据与校验

### 12.5 支付参数构建（Payload）

当选择第三方渠道时，支付 payload 的 `channels` 使用 `thirdParty` 类型，并带上 `couponCode`：

```javascript
{
  orderId: "...",
  payableAmount: 123.45,
  discountAmount: 0,
  channels: [
    { type: "thirdParty", methodId: "dy|mt|offline", amount: 123.45, couponCode: "券码字符串" }
  ]
}
```

### 12.6 发起支付校验

| 校验项 | 说明 |
|-------|------|
| 券码必填 | thirdParty 选择时必须填写券码，否则阻止支付并提示“请输入券码” |
| 禁止组合 | thirdParty 选择时 `channels.length` 必须为 1，且 `type` 必须为 thirdParty |

---

## 验收标准（DoD）

1. 左侧新增“第三方渠道收款”分组，包含抖音/美团/线下三个选项。
2. 选择任一第三方渠道后，“组合收款”不可开启；右侧显示券码输入与“扫码核销”入口。
3. 券码支持手动输入；点击“扫码核销”可调用摄像头识别二维码（支持则自动填充）。
4. thirdParty 场景下点击确认支付，券码为空会被拦截并提示；券码非空可正常发起支付（走现有 mock / API 流程）。
5. 切换回非 thirdParty 方式后，页面行为与现状一致（金额输入/线上二维码/组合收款逻辑不受影响）。
