## 🎨 设计系统引用

## 📊 11.3 订单列表 — 整体布局

| 属性 | 说明 |
|-----|------|
| 布局方式 | **扁平化分组表格**（无折叠/展开交互，所有订单全展开） |
| 订单分组 | 每笔订单为一个视觉区块，包含订单头部行 + 商品明细行（可多行） + 汇总行 |
| 订单间距 | 24px 白色间隔 |
| 核心程度 | **核心** |

### 结构示意

```
┌── 订单头部行 ────────────────────────────────────────┐
│  下单时间：YYYY-MM-DD HH:mm:ss              操作链接   │
├── 商品表头行 ────────────────────────────────────────┤
│  商品 │ 单价 │ 数量 │ 护理师/销售 │ 金额 │ 状态       │
├── 商品行 #1 ────────────────────────────────────────┤
│  [图] 品名      │ ¥xxx │ xN │ 人名/- │ ¥xxx │ 状态标签 │
│       [属性标签] │      │    │       │      │          │
├── 商品行 #2 ────────────────────────────────────────┤
│  [图] 品名      │ ¥xxx │ xN │ 人名/- │ ¥xxx │ (合并)    │
│       [属性标签] │      │    │       │      │          │
├── 订单汇总行 ────────────────────────────────────────┤
│                              订单总额：¥x,xxx         │
└──────────────────────────────────────────────────────┘
```

### 关键设计决策

| 决策 | 说明 |
|------|------|
| **不显示订单编号** | 用户第一眼应看到下单时间和商品，订单编号非首要信息 |
| **不显示品牌/门店** | 品牌名称和下单门店信息不展示（2026-06-07 需求评审确认移除） |
| **多商品订单** | 同一订单多件商品共享订单头部和汇总行，商品行连续排列 |
| **状态跨行合并** | 同一订单的多行共享一个状态标签，居中对齐 |

## 🔄 订单状态流转

### 状态机

```
                      ┌─────────────────────────┐
                      │       待签字              │
                      │   pending_sign (橙色 ●)   │
                      └──────┬──────┬───────────┘
                             │      │
                    客户签字  │      │  退款（客户改变主意，全额/部分）
                             │      │
                             ▼      ▼
                      ┌──────────┐  ┌──────────────────┐
                      │  已完成   │  │  已退款 / 部分退款  │
                      │ completed │  │ refunded /        │
                      │  (绿色)   │  │ partial_refund    │
                      └────┬─────┘  │  (红色)            │
                           │        └──────────────────┘
                  ┌────────┼─────────┐
                  │        │         │
              全额退款  部分退款  (正常完结)
                  │        │
                  ▼        ▼
          ┌──────────┐ ┌──────────┐
          │  已退款   │ │ 部分退款  │
          │ refunded  │ │ partial_  │
          │  (红色)   │ │ refund    │
          └──────────┘ │  (红色)   │
                       └──────────┘
```

### 状态流转规则

| 当前状态 | 可流转至 | 触发条件 |
|----------|----------|----------|
| 待签字 | 已完成 | 客户签字（批量签字或单笔签字） |
| 待签字 | 已退款 | 直接退款（客户改变主意，未签字直接取消） |
| 待签字 | 部分退款 | 直接部分退款 |
| 已完成 | 已退款 | 全额退款 |
| 已完成 | 部分退款 | 仅退部分商品 |

## 🎨 11.10 批量签字弹窗 — Canvas 签名板

| 属性 | 说明 |
|-----|------|
| Canvas ID | `batchSignCanvas` |
| 尺寸 | 480 × 180 px |
| 背景 | `#F5F5F5` |
| 边框 | 虚线 `2px dashed #D1D5DB`, `border-radius: 8px` |
| 默认提示 | "请在此处签名"（灰色居中文字，签字后消失） |
| 核心程度 | **核心** |

### Canvas 交互

| 事件 | 行为 |
|------|------|
| `touchstart` / `mousedown` | 开始绘制（支持触控笔和鼠标） |
| `touchmove` / `mousemove` | 持续绘制，笔触颜色 `#333`，线宽 2px |
| `touchend` / `mouseup` | 结束绘制 |
| 清除重签 | 清空 Canvas + 恢复提示文字 + 禁用确认按钮 |

### 确认签字按钮启用条件

```
确认签字 enabled =
    checkbox.checked === true
    AND
    canvas 签名区域有内容 (drawn === true)
```
